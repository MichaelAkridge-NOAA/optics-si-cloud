🐳 Docker Setup

Docker Engine with persistence helpers for Cloud Workstations

Overview

This guide installs Docker Engine on your Cloud Workstation with helpers for managing image persistence across restarts.

What you'll get:

  • Docker Engine and Docker Compose
  • Log rotation configured
  • Management commands: docker-status, docker-cleanup
  • Image save/load for persistence

Time: ~3 minutes

Step 1: Run the Setup Script

curl -sL https://raw.githubusercontent.com/MichaelAkridge-NOAA/optics-si-cloud-tools/main/scripts/setup_docker.sh | bash

Step 2: Apply Group Changes

After installation, apply the docker group:

newgrp docker

Or log out and back in.

Step 3: Test Docker

docker run hello-world
Success! You should see "Hello from Docker!"

Understanding Persistence

Important: Docker data in /var/lib/docker does NOT persist across Cloud Workstation restarts. Downloaded images and stopped containers will be lost.

Solutions

Option 1: Save images before restart

docker-save-images

Option 2: Use container registries

docker tag myimage gcr.io/PROJECT/myimage
docker push gcr.io/PROJECT/myimage

Management Commands

Check Status

docker-status

Shows version, running containers, images, and disk usage.

Cleanup

docker-cleanup

Removes stopped containers, unused networks, and dangling images.

Save Images (before restart)

docker-save-images

Saves all images to ~/.docker-images/

Load Images (after restart)

docker-load-images

Restores images from ~/.docker-images/

Common Commands

# Run a container
docker run -d -p 8080:80 nginx

# Build from Dockerfile
docker build -t myapp .

# Docker Compose
docker compose up -d
docker compose down

# View logs
docker logs -f container-name
