☁️ GCloud ADC Setup

Fix GCS access issues with Application Default Credentials

The Problem

Cloud Workstation VMs often have limited OAuth scopes. You might see errors like:

403 Forbidden: Provided scope(s) are not authorized

This happens when applications (like Label Studio, Python scripts) try to access GCS.

Common symptom: You can read from GCS but can't write, or vice versa.

The Solution

Application Default Credentials (ADC) let you authenticate with your own user account and full scopes, bypassing VM limitations.

Step 1: Run the Setup Script

curl -sL https://raw.githubusercontent.com/MichaelAkridge-NOAA/optics-si-cloud-tools/main/scripts/setup_gcloud_adc.sh | bash

The script will:

  1. Check your current gcloud configuration
  2. Show VM OAuth scopes (if on GCE)
  3. Check existing ADC credentials
  4. Set up ADC with full scopes

Step 2: Authenticate

When prompted, a browser will open. Sign in with your Google account and grant the requested permissions:

  • Cloud Platform (full access)
  • Cloud Storage (full control)
  • BigQuery

Step 3: Test Access

Verify GCS access:

# List buckets
gsutil ls

# List contents of a bucket
gsutil ls gs://your-bucket-name/

Test in Python:

from google.cloud import storage
client = storage.Client()
for bucket in client.list_buckets():
    print(bucket.name)

Manual Setup (Alternative)

If you prefer to run the command directly:

gcloud auth application-default login \
    --scopes="https://www.googleapis.com/auth/cloud-platform,https://www.googleapis.com/auth/devstorage.full_control,https://www.googleapis.com/auth/bigquery"

How It Works

ADC credentials are stored in:

~/.config/gcloud/application_default_credentials.json

Applications using Google Cloud client libraries automatically find and use this file.

✓ Persists across restarts — The credentials file is in your home directory.

Troubleshooting

Still getting 403 errors?

  • Make sure you authenticated with an account that has access to the bucket
  • Check bucket IAM permissions in the Cloud Console
  • Verify the project: gcloud config get-value project

Revoke and re-authenticate

gcloud auth application-default revoke
gcloud auth application-default login --scopes=...
