🏥 Workstation Health Check

Monitor all services and system resources

Overview

The health check script provides a unified view of all installed services and system resources on your Cloud Workstation.

What it checks:

  • System resources (disk, memory, CPU)
  • Label Studio, JupyterLab, code-server
  • Docker, Python environment
  • Google Cloud (gcloud, ADC)
  • Git & SSH configuration

Run Health Check

curl -sL https://raw.githubusercontent.com/MichaelAkridge-NOAA/optics-si-cloud-tools/main/scripts/workstation_health.sh | bash

Or if cloned locally:

bash scripts/workstation_health.sh

JSON Output

For automation and monitoring tools:

workstation_health.sh --json

Example output:

{
  "timestamp": "2026-03-25T10:30:00",
  "summary": {"passed": 8, "warnings": 1, "failed": 0},
  "resources": {"disk_percent": 45, "memory_percent": 62},
  "services": [
    {"name": "label-studio", "status": "running", "port": "8080"}
  ]
}

Troubleshooting

Service Not Running

# Label Studio
label-studio-restart

# JupyterLab
jupyter-start

# code-server
code-server-start

# Docker
sudo systemctl start docker

High Disk Usage

workstation_cleanup.sh

ADC Not Configured

bash setup_gcloud_adc.sh

Automate Health Checks

Add to cron for regular monitoring:

# Edit crontab
crontab -e

# Hourly health check
0 * * * * ~/scripts/workstation_health.sh --json >> ~/.local/share/health.log
