📓 JupyterLab Setup

Interactive notebooks on Cloud Workstations (port 8888)

Overview

This guide installs JupyterLab with useful extensions in a persistent location.

What you'll get:

  • JupyterLab on port 8888
  • Extensions: Git integration, LSP, widgets
  • Management commands: jupyter-start, jupyter-stop, etc.
  • Pre-installed: pandas, numpy, matplotlib, seaborn, plotly

Time: ~3 minutes

Step 1: Run the Setup Script

curl -sL https://raw.githubusercontent.com/MichaelAkridge-NOAA/optics-si-cloud-tools/main/scripts/setup_jupyter.sh | bash

The script will install JupyterLab and start it automatically.

Step 2: Access JupyterLab

JupyterLab runs on port 8888. Access it via Cloud Workstation port forwarding:

  1. In the Cloud Workstations console, find your workstation
  2. Click on the port forwarding URL for port 8888
  3. Or use: https://8888-YOUR-WORKSTATION.cloudworkstations.dev
No password required — Cloud Workstation IAM handles authentication.

Step 3: Reload Shell (for commands)

source ~/.bashrc

Management Commands

Command Description
jupyter-startStart JupyterLab
jupyter-stopStop JupyterLab
jupyter-statusCheck if running
jupyter-logsView logs

Included Extensions

  • jupyterlab-git — Git integration in sidebar
  • jupyterlab-lsp — Code intelligence (autocomplete, go-to-definition)
  • ipywidgets — Interactive widgets

Enable Auto-Start (Optional)

To start JupyterLab automatically on workstation boot:

nano ~/.bashrc

Find and uncomment the JupyterLab auto-start section near the bottom.

Persistence

✓ Everything persists across restarts!
  • JupyterLab environment: ~/.jupyter-env
  • Configuration: ~/.jupyter
  • Notebooks: Save in ~/
