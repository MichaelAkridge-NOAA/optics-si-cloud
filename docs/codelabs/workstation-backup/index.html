💾 Workstation Backup

Back up your home directory to Google Cloud Storage

Overview

This guide covers backing up your Cloud Workstation to GCS for disaster recovery.

What gets backed up:

  • Configuration files (.bashrc, .gitconfig)
  • SSH keys (~/.ssh)
  • Project files and notebooks
  • Scripts and custom tools

What's excluded:

  • Package caches (pip, npm)
  • Virtual environments (can be recreated)
  • Log files and temp files

Prerequisites

  • GCS bucket with write access
  • ADC configured (run setup_gcloud_adc.sh)
Note: Create a dedicated bucket for backups, e.g., gs://my-project-backups/workstations/

Create a Backup

bash workstation_backup.sh backup gs://your-bucket/workstation-backups

Or download and run:

curl -sL https://raw.githubusercontent.com/MichaelAkridge-NOAA/optics-si-cloud-tools/main/scripts/workstation_backup.sh | bash -s -- backup gs://your-bucket/backups

Backups are named with timestamps: backup-20260325-103045

List Available Backups

bash workstation_backup.sh list gs://your-bucket/workstation-backups

Output:

Available backups:
  backup-20260325-103045  (2026-03-25T10:30:45)
  backup-20260324-140000  (2026-03-24T14:00:00)

Restore from Backup

bash workstation_backup.sh restore gs://your-bucket/workstation-backups/backup-20260325-103045

The script will:

  1. Show backup metadata
  2. Ask for confirmation
  3. Sync files from GCS
After restore: Run source ~/.bashrc and reinstall any virtual environments.

Schedule Automatic Backups

Set up daily backups at 2:00 AM:

bash workstation_backup.sh schedule gs://your-bucket/workstation-backups

View scheduled backups:

crontab -l | grep backup

Best Practices

Storage Location

Use a consistent path:

gs://my-project-backups/workstations/USERNAME/

Retention Policy

Set up lifecycle rules to delete old backups:

# lifecycle.json
{
  "rule": [{
    "action": {"type": "Delete"},
    "condition": {"age": 30}
  }]
}

gsutil lifecycle set lifecycle.json gs://your-bucket

When to Backup

  • Before major software updates
  • Before configuration changes
  • Daily via scheduled backups
