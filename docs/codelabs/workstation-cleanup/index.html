🧹 Workstation Cleanup

Clean logs, caches, and temporary files

Overview

The cleanup script helps free disk space by removing logs, caches, and temporary files. It's safe by default and won't delete user data without confirmation.

Categories cleaned:

  • Package caches (pip, npm, yarn)
  • Application logs (older than 7 days)
  • Python bytecode (__pycache__)
  • Trash folder
  • Docker unused resources

Preview (Dry Run)

See what would be deleted without removing anything:

curl -sL https://raw.githubusercontent.com/MichaelAkridge-NOAA/optics-si-cloud-tools/main/scripts/workstation_cleanup.sh | bash -s -- --dry-run

This shows each category and how much space it uses.

Interactive Cleanup

Run cleanup and choose what to delete:

bash workstation_cleanup.sh

For each category, you'll be prompted:

  pip cache: 245M
    Clean? (y/n)

Automatic Cleanup

Clean everything without prompts:

bash workstation_cleanup.sh --all
Note: This is useful for scripts and automation.

What Gets Cleaned

CategoryLocationSafe?
pip cache~/.cache/pip✓ Always safe
npm cache~/.npm/_cacache✓ Always safe
Python bytecode__pycache__/✓ Always safe
Old logs~/.label-studio/*.log✓ Always safe
Trash~/.local/share/Trash✓ Always safe
Docker unused/var/lib/docker⚠ Review first

What's Never Deleted

  • Source code and projects
  • Virtual environments (preserved)
  • Configuration files
  • SSH keys
  • Git repositories

Schedule Regular Cleanup

Add weekly cleanup to cron:

# Edit crontab
crontab -e

# Weekly cleanup (Sundays at 3 AM)
0 3 * * 0 ~/scripts/workstation_cleanup.sh --all >> ~/.local/share/cleanup.log 2>&1
